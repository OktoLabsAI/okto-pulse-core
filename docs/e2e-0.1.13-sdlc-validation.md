## Remaining Non-Conformities

### Automatic KG consolidation fails

The functional E2E passed, but automatic KG consolidation did not remain healthy.

Final health snapshot:

```text
queue_depth: 0
dead_letter_count: 13
total_nodes: 5
default_score_ratio: 0.0
```

Dead-letter rows failed with:

```text
KGPrimitiveError: commit failed and was compensated:
Binder exception: Trying to insert into an index on table Entity but its extension is not loaded.
```

`okto_pulse_kg_migrate_schema` returned success for the board, but the failed queue rows remained in DLQ.

### DLQ reprocess tool is not visible

Tool discovery still exposed `okto_pulse_kg_dead_letter_list` as read-only and stated that MCP reprocess is deferred. No visible retry/reprocess tool was available during the E2E.

### Consolidated traceability report tool is not visible

No dedicated MCP report was visible for:

```text
ideation -> refinement -> spec -> sprint -> card -> test/bug -> artifacts
```

Traceability had to be reconstructed from `get_ideation_context`, `get_refinement_context`, `get_spec_context`, sprint context, card context, validations, task conclusions, and KG queries.

### Some MCP tool descriptions are stale

Observed description mismatches:

- `okto_pulse_create_card` still says the spec must be `done`, although test cards were accepted after validation and before final spec completion.
- `okto_pulse_create_card` still documents `card_type` as `normal` or `bug`, while `test` is accepted and required for test/regression flows.
- `okto_pulse_submit_task_validation` still says failed validation routes to `not_started`; the desired and observed behavior is to remain in `validation`.
- `okto_pulse_kg_dead_letter_list` still says DLQ is read-only and reprocess is deferred.

### Ideation KB write tool is still not visible

`get_ideation_context` exposes `knowledge_bases`, but no MCP tool was visible to add a KB directly to an ideation. KB usage had to start from refinement/spec.

## Follow-Up Recommendations

1. Fix LadybugDB/KG index extension loading before automatic consolidation writes into indexed node tables.
2. Expose and document a KG DLQ retry/reprocess MCP tool after schema/index fixes.
3. Add a first-class traceability report MCP tool for the full SDLC chain.
4. Update MCP descriptions for `create_card`, `submit_task_validation`, and KG DLQ tools to match actual behavior.
5. Expose ideation KB creation tools or remove `knowledge_bases` from ideation context until write support exists.
6. Keep deterministic KG edge ownership reserved for the worker and document that cognitive agents must use only allowed cognitive edge types.

## Final Assessment

The application SDLC workflow, gates, artifact propagation, task validation, bug/regression handling, and manual KG queryability are working well enough for the validated product flow.

The main blocker is the automatic KG consolidation path. It currently fails for several artifacts with an index-extension error and leaves DLQ entries that cannot be reprocessed through a visible MCP tool.
